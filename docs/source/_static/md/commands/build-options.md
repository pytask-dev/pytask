| Option                                                     | Default                                                                   | Description                                                                                                                                   |
| ---------------------------------------------------------- | ------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| <code>-c, --config FILE</code>                             | -                                                                         | Path to configuration file.                                                                                                                   |
| <code>--capture [fd\|no\|sys\|tee-sys]</code>              | <code>fd</code>                                                           | Per task capturing method.                                                                                                                    |
| <code>--database-url TEXT</code>                           | -                                                                         | Url to the database.                                                                                                                          |
| <code>--debug-pytask</code>                                | <code>false</code>                                                        | Trace all function calls in the plugin framework.                                                                                             |
| <code>--disable-warnings</code>                            | <code>false</code>                                                        | Disables the summary for warnings.                                                                                                            |
| <code>--dry-run</code>                                     | <code>false</code>                                                        | Perform a dry-run.                                                                                                                            |
| <code>--editor-url-scheme TEXT</code>                      | <code>file</code>                                                         | Use file, vscode, pycharm or a custom url scheme to add URLs to task ids to quickly jump to the task definition. Use no_link to disable URLs. |
| <code>--explain</code>                                     | <code>false</code>                                                        | Explain why tasks need to be executed by showing what changed.                                                                                |
| <code>-f, --force</code>                                   | <code>false</code>                                                        | Execute a task even if it succeeded successfully before.                                                                                      |
| <code>--hook-module TEXT</code>                            | -                                                                         | Path to a Python module that contains hook implementations.                                                                                   |
| <code>--ignore TEXT</code>                                 | -                                                                         | A pattern to ignore files or directories. Refer to 'pathlib.Path.match' for more info.                                                        |
| <code>-k EXPRESSION</code>                                 | -                                                                         | Select tasks via expressions on task ids.                                                                                                     |
| <code>--log-cli / --no-log-cli</code>                      | <code>--no-log-cli</code>                                                 | Enable live log display during task execution.                                                                                                |
| <code>--log-cli-date-format TEXT</code>                    | -                                                                         | Log date format used by the logging module for live logs.                                                                                     |
| <code>--log-cli-format TEXT</code>                         | -                                                                         | Log format used by the logging module for live logs.                                                                                          |
| <code>--log-cli-level LEVEL</code>                         | -                                                                         | CLI logging level.                                                                                                                            |
| <code>--log-date-format TEXT</code>                        | <code>%H:%M:%S</code>                                                     | Log date format used by the logging module.                                                                                                   |
| <code>--log-file TEXT</code>                               | -                                                                         | Path to a file where logging will be written.                                                                                                 |
| <code>--log-file-date-format TEXT</code>                   | -                                                                         | Log date format used by the logging module for the log file.                                                                                  |
| <code>--log-file-format TEXT</code>                        | -                                                                         | Log format used by the logging module for the log file.                                                                                       |
| <code>--log-file-level LEVEL</code>                        | -                                                                         | Log file logging level.                                                                                                                       |
| <code>--log-file-mode [w\|a]</code>                        | <code>w</code>                                                            | Log file open mode.                                                                                                                           |
| <code>--log-format TEXT</code>                             | <code>%(levelname)-8s %(name)s:%(filename)s:%(lineno)d %(message)s</code> | Log format used by the logging module.                                                                                                        |
| <code>--log-level LEVEL</code>                             | -                                                                         | Level of messages to catch/display. Not set by default, so it depends on the logger configuration.                                            |
| <code>-m MARKER_EXPRESSION</code>                          | -                                                                         | Select tasks via marker expressions.                                                                                                          |
| <code>--max-failures FLOAT RANGE</code>                    | <code>inf</code>                                                          | Stop after some failures.                                                                                                                     |
| <code>--n-entries-in-table INTEGER RANGE</code>            | <code>15</code>                                                           | How many entries to display in the table during the execution. Tasks which are running are always displayed.                                  |
| <code>--pdb</code>                                         | <code>false</code>                                                        | Start the interactive debugger on errors.                                                                                                     |
| <code>--pdbcls module_name:class_name</code>               | -                                                                         | Start a custom debugger on errors. For example: --pdbcls=IPython.terminal.debugger:TerminalPdb                                                |
| <code>-s</code>                                            | <code>false</code>                                                        | Shortcut for --capture=no.                                                                                                                    |
| <code>--show-capture [no\|stdout\|stderr\|log\|all]</code> | <code>all</code>                                                          | Choose which captured output should be shown for failed tasks.                                                                                |
| <code>--show-errors-immediately</code>                     | <code>false</code>                                                        | Show errors with tracebacks as soon as the task fails.                                                                                        |
| <code>--show-locals</code>                                 | <code>false</code>                                                        | Show local variables in tracebacks.                                                                                                           |
| <code>--show-traceback / --show-no-traceback</code>        | <code>--show-traceback</code>                                             | Choose whether tracebacks should be displayed or not.                                                                                         |
| <code>--sort-table / --do-not-sort-table</code>            | <code>--sort-table</code>                                                 | Sort the table of tasks at the end of the execution.                                                                                          |
| <code>--strict-markers</code>                              | <code>false</code>                                                        | Raise errors for unknown markers.                                                                                                             |
| <code>--trace</code>                                       | <code>false</code>                                                        | Enter debugger in the beginning of each task.                                                                                                 |
| <code>-v, --verbose INTEGER RANGE</code>                   | <code>1</code>                                                            | Make pytask verbose (>= 0) or quiet (= 0).                                                                                                    |
| <code>-x, --stop-after-first-failure</code>                | <code>false</code>                                                        | Stop after the first failure.                                                                                                                 |
| `-h, --help`                                               | -                                                                         | Show this message and exit.                                                                                                                   |
